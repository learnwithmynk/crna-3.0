## Responsive Behavior

- **Mobile (< 768px)**
  - Cards stack vertically
  - Buttons stack on smaller screens
  - Icon + text layout for settings items
  - Touch-friendly targets (44px minimum)

- **Tablet (768px - 1024px)**
  - Cards maintain spacing
  - Buttons stay inline where possible
  - Optimal reading width maintained

- **Desktop (> 1024px)**
  - Max width container (max-w-4xl)
  - Centered layout
  - All features fully visible
  - Hover states on interactive elements

## Interaction States

1. **Hover**: Slight opacity change on buttons
2. **Active**: Scale down slightly (Apple-style press)
3. **Focus**: Blue ring around focused elements
4. **Disabled**: 50% opacity, pointer-events-none
5. **Loading**: Button shows loading state with disabled appearance
